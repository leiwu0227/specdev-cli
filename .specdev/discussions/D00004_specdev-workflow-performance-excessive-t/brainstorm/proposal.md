# Proposal: Cut SpecDev Workflow Test Overhead

SpecDev's `specdev implement` and `specdev reviewloop` feel slow because the workflow forces full-suite test runs ~13 times per typical 9-task assignment, agents over-produce test cases beyond what the plan requires, and (in this repo specifically) `npm test` is a chain of 28 sequential `npm run` calls with no parallelism. The workflow-side problem ships to every SpecDev user via `templates/.specdev/skills/core/`, so the fix benefits all consumers, not just this repo.

The proposal has three coordinated layers. **(A) Workflow skills** — edit `test-driven-development/SKILL.md` and `implementing/SKILL.md` so RED/GREEN/REFACTOR pass a scoped test command to `verify-tests.sh` (the parameter already exists, no script change needed), drop the per-batch full-suite run, and keep only the end-of-phase full run. **(B) This repo's test harness** — replace the 28-script chain with `node --test --test-concurrency=N`, quarantine the known-hang `test-reviewloop-command.js`, and audit for shared state. **(C) Test-case budget** — require `breakdown/plan.md` to enumerate tests per task with a default of ≤ 1 new test per task / ≤ 5 per assignment, and have the implementation reviewer count and flag deviations. Net effect for a 9-task assignment: ~13 full-suite runs → 1 full + ~9 scoped, plus a bounded, smaller test suite that stays fast forever.
