This is a findings-only audit. No fixes are applied. Each finding cites a
specific path, tags type (`legacy / contradiction / inconsistency /
duplication / drift`) and severity (`high / med / low`), and gives a
suggested action specific enough that a follow-up assignment could be opened
from it. Severity labels from per-area inventories that used
`major/moderate/minor` have been normalized to `high/med/low`.

**Top 5 highest-impact items**

1. **`.specdev/` runtime is multiple patches behind `templates/.specdev/`** (drift, high). `_main.md`, `_index.md`, `_guides/workflow.md`, `_guides/assignment_guide.md`, and `_templates/gate_checklist.md` in the installed runtime still describe the old 4-phase workflow with a required Summary phase writing `capture/*-diff.md`, no `specdev next --json`, and required (not optional) Knowledge Capture. Templates have already moved to 3 phases + optional knowledge capture + `workflow.yaml`/`specdev next --json` overlay. Every Claude/Codex session in this repo reads the drifted files. **Action:** `specdev update` (or sync the five files manually) and verify with a `diff -u templates/.specdev/<path> .specdev/<path>` pass.
2. **`docs/plans/` (35 files) is a fossil layer** (legacy, high). None of `src/`, `bin/`, `templates/.specdev/`, or installed `.specdev/` reference any file under `docs/plans/`. Most describe removed commands (`work`/`check`/`ponder`/`remind`/`autoloop`, skills `executing`/`orientation`/`planning`/`scaffolding-lite/full`) or design states already superseded. **Action:** archive whole-sale to `.specdev/knowledge/_archive/plans/` or delete; rely on `.specdev/assignments/<id>/brainstorm/design.md` + `breakdown/plan.md` going forward.
3. **Legacy `distill` chain still wired across the codebase** (legacy, high). `src/commands/distill.js` and `distill-done.js` advertised "Legacy helper" in help text; help/QUICKSTART/CHANGELOG promise `distill workflow`/`distill project` shapes that the CLI does not provide; `continue.js` still emits a `distill_pending` suggestion; utility helpers in `src/utils/scan.js` (`readKnowledgeBranch`, `mark/readProcessedCaptures`) exist only for distill; `tests/test-knowledge.js` keeps the legacy `--assignment=<name>` form alive. **Action:** decide keep-or-drop. If drop: remove the two commands, their dispatcher branches, the `continue.js` block, the scan helpers, the QUICKSTART references, and the CHANGELOG entries.
4. **`scripts/verify-gates.sh` is dead but quoted from three places** (legacy, med). The script asserts V3 artifacts (`scaffold/_architecture.md`, legacy `implementation.md` TDD tables, `review_request.json` with `gate_3/gate_4`) that no current command produces. No caller exists in `package.json`, `hooks/`, `src/`, or `tests/`, but `templates/.specdev/_templates/review_report_template.md` and `.specdev/_templates/review_report_template.md` still instruct reviewers to paste its output, and three `docs/plans/` files reference it. **Action:** delete the script and remove the "Pre-flight (verify-gates.sh)" block in both review_report_template.md copies (or rewrite both around `scripts/verify-assignment-schema.js`, which the maintained test suite actually exercises).
5. **00014 assignment is in a contradictory state and 00022 is uncaptured** (contradiction, high). 00014_feature_agent-friendly-workflow has `implementation/progress.json`, multiple Codex round logs, and `review/implementation-feedback.md`, but its `status.json` records only `brainstorm_approved: true`, and it is missing from `assignment_progress.md`, `feature_descriptions.md`, and `.processed_captures.json`. 00022_refactor_thin-runtime-guidance has both gates approved but is missing from all four state files (`assignment_progress.md`, `feature_descriptions.md`, `.processed_captures.json`, `working_memory.md`). **Action:** decide 00014's true status (abandoned vs. completed-uncaptured) and reconcile; run capture/distill for 00022 (or mark intentionally skipped), then `specdev memory refresh`.
