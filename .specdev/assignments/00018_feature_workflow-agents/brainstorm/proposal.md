# Proposal: Introduce Agents as a SpecDev Workflow Primitive

SpecDev's workflow today is built from two primitives: **skills** (prose protocols the host coding agent reads and follows) and **scripts** (deterministic mechanics). One workflow step — `specdev reviewloop` — already breaks this duality by calling an external reasoning CLI as a tool, but the abstraction is private to review. This assignment introduces **agents** as a first-class third primitive: named, contract-bound reasoning subroutines the workflow (or the host agent) can call, distinct from scripts (deterministic) and skills (prose for the host).

We will build one agent end-to-end — a **researcher** that takes a topic + scope and produces structured research notes across the local repo, `.specdev/` knowledge, and the public web. The researcher will be invokable via `specdev research "<topic>"` and recommended in skill hints (e.g., brainstorming) when evidence is thin. Alongside it we will ship a path-based `agent-runner` utility designed from day one to plausibly subsume the existing reviewer-runner later — but **without** touching reviewloop in this assignment. The agent spec format is a portable Markdown Agent Spec (`agent.md`) with advisory frontmatter and a strict sidecar JSON Schema for output validation. The next three target agents (reviewloop migration, workflow-knowledge distill, codebase-knowledge distill) are documented as roadmap so the abstraction is designed against real upcoming consumers, not speculation.
